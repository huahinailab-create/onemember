## Purpose

This document defines the core values that guide how everyone at OneMember — people and AI systems — makes decisions when rules don't give a clear answer.

Values are not aspirations. They are the actual behaviours that define the culture today, observable in the decisions already being made.

---

## Core Values

> *(To be finalised in Sprint AI-02B. Candidate values based on decisions already made:)*

### Simplicity First
If a merchant needs training to use it, it is too complex. If a developer needs a diagram to understand the code, it is too abstract. Reduce until it can't be reduced further without breaking.

### Merchant Sovereignty
Merchants own their customer relationships. Every product decision should increase that ownership, never reduce it. We do not intermediate the merchant-customer relationship.

### Privacy by Design
Customer data is never shared without explicit, specific consent. Privacy is not a compliance checkbox — it is a core product feature and a competitive advantage.

### Quality Without Compromise
No feature ships with failing tests. No bug is hidden behind a workaround. No technical debt is accumulated silently. We fix things properly or we flag them openly.

### Honest Communication
Between the team, with users, in code comments, and in documentation. We say what is true, not what is convenient. We flag problems early, not after they compound.
