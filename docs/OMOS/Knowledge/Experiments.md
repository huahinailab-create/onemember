## Purpose

Records of product experiments, A/B tests, and hypothesis-driven feature tests — what was tested, what was measured, and what was learned.

---

## How to Use This File

Document hypothesis-driven tests here. A good experiment has:
1. A clear hypothesis ("we believe X will cause Y")
2. A measurement method (how we will know if it worked)
3. A defined outcome (what we will do if it works / fails)

Format:
```
### EXP-[NNN] — [Title]
**Hypothesis:** We believe [X] will cause [Y] because [reason]
**Date Started:** YYYY-MM-DD
**Date Ended:** YYYY-MM-DD or Ongoing
**Measurement:** [What metric we tracked and how]
**Result:** [What actually happened]
**Decision:** [What we did as a result — build it / park it / iterate]
```
